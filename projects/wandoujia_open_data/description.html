Introduction

This data set is user behavioral profile of using Android applications (apps for short). The dataset is collected from about 100,000 Chinese users, ranging from 1st May, 2014 to 30th September, 2014. Our dataset is provided by Wandoujia, which is one of the leading Android marketplaces in China. Wandoujia presents a native Android app which provides management tools to search, download, update, and uninstall apps on their Android devices. In addition, the Wandoujia management app can also record the daily network activities such as the traffic volume and connection time of Wi-Fi/cellular per app, and distinguishes the traffic and time in terms of foreground and background, respectively. The Wandoujia management app generates the daily network statistics only when the users explicitly grant, and all the users' privacy are hidden in this data set.

The data set contains 3 parts:

  • Management Activity Data: The app management activities include downloading, updating, and uninstalling apps. Each entry in the dataset represents a management action that is done via the Wandoujia management app.

  • Network Trace Data: If the user allows the Wandoujia management app to collect the network activity data, the Wandoujia management app can make daily network statistics of each app. The Wandoujia management app distinguisheshes the network trace data in terms of both Wi-Fi and cellular (2G/3G/LTE), respectively. The Wandoujia management app does not record the detail of user's network usage at session level. Instead, the management app summarizes the total daily traffic and access time of an app by examining flows at the TCP-level. In particular, the traffic and access time generated from foreground and background are treated separately. The Wandoujia management app determines whether an app is running at foreground or background by probing the Android system stack for every 2 seconds. If an app is on the top of the Android system stack in a probe, it will be considered as running at foreground in the past 2 seconds. So in the network activity data there are 8 aspects of information in total, i.e.,

    2 metrics (access time and traffic) * 2 modes (Wi-Fi and Cellular) * 2 states (foreground and background).

  • Device Model Price: The Wandoujia management app also records each user's device model type, e.g., Samsung Galaxy Note 2, Samsung S3, etc. This data set provides the price of top 500 most popular device models.

Anonymization

To protect the privacy of users, app developers, and phone manufactures, we anonymize all the specific information of users' ID, apps' package names, and device models' names. Instead, we use three groups of simple identifiers: "aXX", "uXX", "dXX" to represent apps, users, and device models. "XX" here means a number.

For example, in this data set there are 100,000 users in total. The names of the users are from "u1" to "u100000". For apps and device models, the identifiers are similar.

Data Description

Management Activity Data

The management activity data has 4 fields, separated by tab. The meaning and detailed explanation of each field are listed as follows:

Package Name The app's identifier.
User ID The user's identifier.
Timestamp The standard UNIX timestamp which represents the time this action takes.
Action Type The type of this action. "d" for download, "u" for uninstallation, and "p" for update.

Network Trace Data

The network trace data has 12 fields, separated by tab. The meaning and detailed explanation of each field are listed as follows:

Package Name The app's identifier.
User ID The user's identifier.
Device Model The device model this user uses in this day.
Date The date of this data entry.
B-Cellular Traffic Background cellular traffic usage. (Byte)
B-Cellular Time Background cellular network connection time. (ms)
F-Cellular Traffic Foreground cellular traffic usage. (Byte)
F-Cellular Time Foreground cellular network connection time. (ms)
B-WiFi Traffic Background Wi-Fi traffic usage. (Byte)
B-WiFi Time Background Wi-Fi network connection time. (ms)
F-WiFi Traffic Foreground Wi-Fi traffic usage. (Byte)
F-WiFi Time Foreground Wi-Fi network connection time. (ms)
